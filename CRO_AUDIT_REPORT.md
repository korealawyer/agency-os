## 🚨 문제 #1: 모바일 반응형 전무 — 광고비의 블랙홀

### [문제점 비판]

**이것 하나만으로 광고 집행이 불가능합니다.** 네이버 검색광고 유입의 60~70%는 모바일입니다. 그런데 이 사이트는 모바일에서 **완전히 깨집니다.**

| 페이지 | 모바일 상태 | 심각도 |
|--------|-----------|:------:|
| 랜딩 | 네비게이션 텍스트 줄바꿈+겹침, "기능/가격/데모/ROI계산기/로그인" 한 줄에 억지로 표시 | 🔴 |
| 가격 | 5개 플랜 카드가 가로 나열 → 글자 찌그러짐, CTA 버튼 탭 불가 | 🔴 |
| 대시보드 | 사이드바 미분리, 데이터 테이블 가로 오버플로우 | 🔴 |
| 키워드 관리 | 계정명 "A 법률사무소"가 세로로 한 글자씩 표시 | 🔴 |

### [원인 분석]

- **미디어 쿼리(Media Query) 미적용**: CSS에 `@media` 분기점이 거의 없어 모든 요소가 데스크탑 레이아웃 그대로 축소됩니다
- **네비게이션에 햄버거 메뉴 부재**: 7개의 메뉴 항목이 375px 폭에 강제 배치 → 줄바꿈 + 텍스트 겹침
- **Flex/Grid 반응형 미설정**: 가격 카드 5열 → 모바일에서도 5열 유지 → 카드당 60px 폭

> 모바일 유저는 **3초 이내에** 페이지의 전문성을 판단합니다. 레이아웃이 깨진 사이트를 본 순간 "이 서비스가 내 광고도 이렇게 관리하겠지"라는 부정적 **첫인상 편향(Primacy Effect)**이 발동합니다. **반송률(Bounce Rate)** 90% 이상 예상.

### [개선 방안 — 3가지 즉시 실행]

1. **햄버거 메뉴 도입** (768px 이하)
   - 로고 + 햄버거 아이콘 + "14일 무료 체험" CTA만 노출
   - 나머지 메뉴는 슬라이드 오버레이로 처리

2. **가격 카드 세로 스택** (768px 이하)
   - `grid-template-columns: 1fr` 적용
   - "Growth (추천)" 카드만 먼저 노출 → 나머지는 "더보기"로 접기

3. **데이터 테이블 → 카드 뷰 전환**
   - 키워드 관리, 캠페인 테이블을 모바일에서 카드형 리스트로 변환
   - 핵심 지표(키워드명, 입찰가, 순위)만 노출, 상세는 펼치기

## 🚨 문제 #2: 전환 퍼널(CTA)의 불일치와 단절

### [문제점 비판]

유저가 "14일 무료 체험"을 클릭하면 무슨 일이 일어나야 할까요? **아무 일도 일어나지 않습니다.** 현재 CTA 버튼들은:
- 랜딩 히어로의 "14일 무료 체험 →" → 행선지 불명
- 가격 페이지의 "14일 무료 체험" → 행선지 불명
- 헤더의 "14일 무료 체험" → 행선지 불명

**돈을 써서 데려온 고객이 "구매 버튼"을 눌렀는데 아무 반응이 없는 것은 매장에서 계산대를 없앤 것과 같습니다.**

또한 CTA 디자인이 페이지마다 다릅니다:

| 위치 | CTA 스타일 | 문제 |
|------|----------|------|
| 헤더 | 진한 남색 pill 버튼 | — |
| 히어로 | 보라색 그라데이션 + 화살표 | 헤더와 불일치 |
| 가격 카드 | 진한 남색 사각 버튼 | 히어로와 불일치 |
| 데모 하단 | 보라색 그라데이션 | 가격과 불일치 |

### [원인 분석]

- **전환 경로(Conversion Path) 미설계**: 회원가입 페이지가 존재하지 않아 CTA가 행선지를 잃은 상태
- **디자인 시스템에 CTA 표준 부재**: 컴포넌트별로 버튼 스타일이 제각각
- 유저 심리: "버튼을 눌렀는데 아무 일도 안 일어나면?" → **학습된 무기력(Learned Helplessness)** 발동 → 재클릭 의지 소멸 → **CVR** 0%

### [개선 방안]

1. **CTA 색상/스타일 단일화**: 전 사이트에서 보라색 그라데이션 + 화살표 아이콘으로 통일 (히어로 버전이 가장 시인성 높음)
2. **CTA 클릭 → 회원가입 모달 즉시 표시**: 별도 페이지 이동 없이 모달로 이메일만 입력받는 최소 가입 폼 (이름 + 이메일 + 비밀번호, 3개 필드)
3. **가격 카드별 CTA 카피 차별화**: "무료로 시작하기" / "14일 무료 체험" / "상담 신청" → 각 플랜의 행동유도가 명확하게 다른 것은 좋으나, **클릭 후 실제 전환이 일어나야** 의미 있음

## 🚨 문제 #3: 신뢰 요소(Social Proof) 부재

### [문제점 비판]

**"320% ROAS를 달성했다고요? 누가요? 증거는요?"**

랜딩 페이지 히어로에 "₩45.2M 광고비, 320% ROAS" 등 인상적인 수치가 나옵니다. 하지만 이것이:
- 실제 고객 데이터인지
- 시뮬레이션 수치인지
- 목표치인지

**전혀 명시되어 있지 않습니다.** B2B SaaS에서 가장 중요한 것은 **신뢰**입니다. 검색광고로 유입된 대행사 대표/마케터는 숫자에 매우 예민합니다. 근거 없는 수치 나열은 오히려 **불신**을 키웁니다.

데모 페이지 하단에 "고객 사례" 3개가 있지만:
- 회사 로고 없음
- 담당자 이름/직책 없음
- "A 마케팅 대행사", "B 퍼포먼스 에이전시" — 가짜 느낌

### [원인 분석]

- **사회적 증거(Social Proof)의 부재**: Cialdini의 설득 원리 중 가장 강력한 '사회적 증거'가 활용되지 않음
- B2B 구매자는 **동종 업계의 성공 사례**에 5배 더 높은 신뢰를 보임
- 현재 상태는 "주장만 있고 증거는 없는 세일즈 피치" → **체류시간** 단축, **이탈률** 상승

### [개선 방안]

1. **히어로 섹션 바로 아래에 고객 로고 배너** 추가: "N개 대행사가 Agency OS로 전환 중" + 로고 5~8개 가로 슬라이드
2. **구체적 사례 연구(Case Study) 카드**: "OO법률대행사 — ROAS 320% 달성, 리포트 작업 96% 감소" + 실제 담당자 인용문 + 사진
3. **실시간 사용 현황 카운터**: "현재 OO개 대행사가 OO개 계정을 관리 중" — 동적 수치로 신뢰감 증폭

## ⚠️ 문제 #4: ROI 계산기의 비현실적 수치 — 신뢰 파괴

### [문제점 비판]

ROI 계산기는 **전환의 핵심 무기**가 될 수 있는 훌륭한 도구입니다. 하지만 현재 **수치가 비현실적**이어서 오히려 역효과를 냅니다.

- 관리 계정 15개, 월 광고비 1억 설정 시: **하루 절감 시간 7.2시간** → 하루 근무 시간이 8시간인데, 절감 시간이 7.2시간? 
- 계정 수를 더 올리면 "하루 절감 47.8시간" — **물리적으로 불가능한 수치**

### [원인 분석]

- 절감 시간 계산 로직이 **계정 수 × 업무당 절감시간** 단순 곱셈 → 상한선 미설정
- 대행사 실무자는 **숫자의 신빙성**에 민감 — 비현실적 수치 = "이 서비스도 뻥이겠네"
- ROI 계산기의 목적은 "와, 이만큼 절약된다고?" 가 아니라 "합리적으로 이만큼 절약되겠구나" 라는 **논리적 확신**

### [개선 방안]

1. **시간 절감 상한선 설정**: 1인 기준 하루 최대 4시간, 팀 기준 명시 ("5인 팀 기준 일 35시간")
2. **단위 변경**: "하루 절감" → "**월간 절감**"으로 바꾸고, 원화 환산 강조 (시급 ₩25,000 기준 → 월 ₩360만 절감)
3. **결과 카드에 CTA 직접 연결**: 추천 플랜 "Growth" 카드에 "**이 플랜으로 시작하기 →**" 버튼 삽입 → 계산기 → 전환의 원스텝 퍼널 완성

## ⚠️ 문제 #5: 가격 페이지의 선택 과부하 (Choice Overload)

### [문제점 비판]

5개 플랜은 **너무 많습니다.** 유저가 "수동적 탈락 없이 선택"하도록 돕는 것이 가격 페이지의 존재 이유입니다. 현재:

- Personal(무료) / Starter(₩30만) / **Growth(₩60만)** / Scale(₩100만) / Enterprise(협의)
- "Growth"에 "⭐ 추천" 뱃지가 있지만, **시각적 차별화가 약합니다** — 테두리 색상만 다를 뿐 카드 크기/배경이 동일

### [원인 분석]

- **Hick의 법칙**: 선택지가 많을수록 결정 시간이 기하급수적으로 증가 → **결정 피로(Decision Fatigue)** → 이탈
- B2B SaaS에서 첫 전환은 "무료 체험"이므로, 플랜 비교에 시간을 쓰게 만드는 것은 불필요한 마찰
- 추천 플랜의 시각적 강조가 약하면 **앵커링 효과(Anchoring Effect)** 실패

### [개선 방안]

1. **3개 플랜으로 축소**: Starter / **Growth (추천)** / Enterprise — 나머지는 "모든 플랜 비교" 링크로 하단 이동
2. **Growth 카드를 20% 더 크게** + 배경 그라데이션 + "가장 인기" 리본 → 시각적 앵커링 강화
3. **"어떤 플랜이 맞나요?" 퀴즈형 CTA** 추가: 관리 광고비 입력 → 자동 추천 → 마찰 없는 의사결정

## ⚠️ 문제 #6: 데모/제품 투어의 정적 경험

### [문제점 비판]

데모 페이지가 **정적 텍스트 + 아이콘 나열**입니다. 동영상 영역에 재생 버튼이 있으나 **실제 영상이 없습니다.** "3분 만에 체험하세요"라고 해놓고, 클릭하면 아무 일도 안 일어납니다.

4가지 핵심 기능 설명은:
- 통합 대시보드 → 텍스트 설명 + 정적 아이콘
- AI 자동 입찰 → 텍스트 설명 + 정적 아이콘
- 리포트 자동화 → 텍스트 설명 + 정적 아이콘
- 자동화 설정 → 텍스트 설명 + 정적 아이콘

**검색광고로 "네이버 광고 자동화 툴"을 검색해서 들어온 유저가 원하는 것은 "보는 것"이 아니라 "만져보는 것"입니다.**

### [원인 분석]

- **인터랙티브 데모 부재**: 옆 정적 이미지만으로는 "Aha Moment" 도달 불가
- 유저 심리: "직접 써봐야 안다" → 정적 페이지는 **체류시간**만 낭비하고 전환으로 연결되지 않음
- 실제 대시보드는 훌륭하게 만들어져 있는데, 데모에서 이것을 보여주지 않는 것은 **최대의 낭비**

### [개선 방안]

1. **실제 대시보드 스크린 GIF/영상 삽입**: 각 기능 설명 옆에 실제 동작 영상 → "어, 이거 진짜 되네!" 반응 유도
2. **인터랙티브 샌드박스 도입**: 실제 대시보드의 읽기 전용(Read-only) 버전을 제공 → 로그인 없이 클릭/탐색 가능
3. **"지금 바로 체험" CTA를 데모 계정 자동 로그인으로 연결**: 현재 로그인 페이지에 demo credentials(admin@agency.com / password)가 표시되어 있으므로, 원클릭 데모 로그인 구현

### 📍 1. 대시보드 홈 (`/dashboard`)

**작동 현황 ✅:**
- KPI 카드 6개 (총 광고비 ₩45.2M, 평균 ROAS 320%, 노출수, 전환수, 클릭수, 활성 키워드) — 전일 대비 증감율 포함

- 일별 ROAS 추이 차트: 마우스 호버 시 툴팁 표시 (**정상**)
- 계정 현황 테이블: 행 클릭 시 계정 상세 페이지로 이동 (**정상**)
- Empty State 미리보기 체크박스 정상 동작

**문제점 🔴:**
- "Empty State 미리보기" 체크박스가 프로덕션 UI에 노출 — **개발자 전용 요소**로 반드시 숨겨야 함
- KPI 카드 클릭 시 상세로 드릴다운 불가 — 핵심 지표를 클릭했을 때 해당 상세 보고서로 연결되어야 함
- 모든 데이터가 하드코딩된 Mock — API 연동 전이므로 변경 불가 상태

### 📍 2. 캠페인 관리 (`/dashboard/campaigns`)

**작동 현황 ✅:**
- 좌측 계정 트리: 광고주별 클릭 시 해당 계정 캠페인만 필터링 (**정상**)
- 상태 필터 탭: 전체/활성/일시정지/종료 탭 정상 전환 (**정상**)
- 검색 기능: 영문 입력 시 필터링 작동 (**정상**)
